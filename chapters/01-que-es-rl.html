¿Qué es el aprendizaje por refuerzo?

El aprendizaje por refuerzo (Reinforcement Learning, RL) es la rama de la inteligencia artificial que estudia cómo un agente aprende a tomar decisiones probando cosas y observando las consecuencias. No le decimos qué hacer: le dejamos actuar, le damos recompensas, y él descubre solo una estrategia que las maximice.

La idea, con una analogía

Piensa en cómo aprende un niño a montar en bicicleta. Nadie le da una fórmula. Prueba, se tambalea, a veces se cae (mala experiencia) y a veces avanza unos metros (buena experiencia). Con el tiempo, su cuerpo ajusta lo que hace para tener más experiencias buenas y menos malas. Eso es exactamente el aprendizaje por refuerzo.

🧠 Intuición

En RL no aprendemos de ejemplos etiquetados («esto es un gato, esto es un perro»), como en el aprendizaje supervisado. Aprendemos de consecuencias: hacemos algo, el mundo responde con una señal numérica de recompensa, y usamos esa señal para actuar mejor la próxima vez.

El bucle agente–entorno

Todo el RL gira en torno a un bucle que se repite en el tiempo. En cada instante $t$:

  1. El agente observa el estado $S_t$ del entorno (¿dónde estoy?).
  2. Elige una acción $A_t$ (¿qué hago?).
  3. El entorno responde con una recompensa $R_{t+1}$ y un nuevo estado $S_{t+1}$.
  4. Vuelta a empezar.

El agente persigue un objetivo: no maximizar la recompensa inmediata, sino la recompensa acumulada a largo plazo. A veces conviene sacrificar premio ahora para ganar mucho más después (estudiar hoy para aprobar mañana).

🔑 Vocabulario esencial

Estos cinco términos aparecerán en todo el manual. Vale la pena fijarlos ya:

  • Agente: quien decide y aprende.
  • Entorno: todo lo demás; responde a las acciones.
  • Estado $S_t$: la información disponible para decidir.
  • Acción $A_t$: lo que el agente puede hacer.
  • Recompensa $R_{t+1}$: la señal numérica que define qué es «bueno».

Un primer vistazo a las matemáticas

El objetivo del agente se puede escribir con precisión. Definimos el retorno $G_t$ como la suma de recompensas futuras, dando un poco menos de peso a las que llegan más tarde mediante un factor de descuento $\gamma \in [0, 1]$:

➗ El objetivo formal

$$G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}$$

El agente busca una política $\pi$ (una forma de elegir acciones) que maximice el retorno esperado $\mathbb{E}_\pi[G_t]$. El descuento $\gamma$ codifica «cuánto me importa el futuro»: con $\gamma=0$ solo cuenta el premio inmediato; con $\gamma \to 1$ el agente es muy paciente.

No te preocupes si aún no ves de dónde sale todo esto: en el capítulo 4 lo construiremos con calma. Por ahora basta con la idea: maximizar la recompensa acumulada y descontada.

Tu primer agente, en Python

Vamos a ver el bucle en acción con el ejemplo más simple posible: un robot en un pasillo de 5 casillas. Empieza a la izquierda (casilla 0) y la meta está a la derecha (casilla 4). Puede ir a izquierda o derecha. Cada paso cuesta $-1$ (queremos llegar rápido) y alcanzar la meta da $+10$.

Compararemos dos «agentes»: uno que actúa al azar y otro con una política fija sensata (ir siempre a la derecha). Todavía no hay aprendizaje —eso empieza en el capítulo 2—, pero verás el bucle y la recompensa.

pasillo.py 
import numpy as np

# --- El entorno: un pasillo de 5 casillas [0, 1, 2, 3, 4] ---
N = 5
META = N - 1

def paso(estado, accion):
    """Devuelve (nuevo_estado, recompensa, terminado)."""
    if accion == "derecha":
        estado = min(estado + 1, META)
    else:  # "izquierda"
        estado = max(estado - 1, 0)
    if estado == META:
        return estado, +10, True      # llegar a la meta
    return estado, -1, False          # cada paso cuesta

def ejecutar_episodio(politica, semilla):
    rng = np.random.default_rng(semilla)
    estado, total, terminado, pasos = 0, 0, False, 0
    while not terminado and pasos < 50:
        accion = politica(estado, rng)
        estado, r, terminado = paso(estado, accion)
        total += r
        pasos += 1
    return total, pasos

# --- Dos políticas (formas de decidir la acción) ---
def politica_azar(estado, rng):
    return rng.choice(["izquierda", "derecha"])

def politica_derecha(estado, rng):
    return "derecha"

# --- Comparamos ambas en 500 episodios ---
for nombre, pol in [("Al azar", politica_azar), ("Siempre derecha", politica_derecha)]:
    recompensas = [ejecutar_episodio(pol, semilla=i)[0] for i in range(500)]
    print(f"{nombre:16s} -> recompensa media = {np.mean(recompensas):6.2f}")

Pulsa ▶ Ejecutar (la primera vez tarda unos segundos en descargar el intérprete de Python). Verás que la política sensata obtiene una recompensa mucho mayor que la aleatoria. La pregunta central del resto del manual es: ¿cómo puede el agente descubrir esa buena política por sí solo, sin que se la digamos?

🧪 Experimenta

Cambia la recompensa de cada paso de $-1$ a $0$ y vuelve a ejecutar. ¿Por qué ahora la política aleatoria ya no parece tan mala en recompensa total? (Pista: sin castigo por paso, tardar no penaliza.) Jugar con las recompensas es diseñar el problema.

🔑 Resumen del capítulo
  • El RL aprende de la interacción: acción → recompensa → mejor decisión.
  • El bucle es agente ↔ entorno, con estados $S_t$, acciones $A_t$ y recompensas $R_{t+1}$.
  • El objetivo es maximizar el retorno $G_t$: recompensa futura acumulada y descontada por $\gamma$.
  • Una política $\pi$ es la estrategia para elegir acciones. Aprender = mejorar la política.
